# Glosario

Términos clave relacionados con la ciberseguridad y la normativa legal, explicados
en euskera y con su equivalente en castellano.

---

| Euskara | Castellano | Explicación |
|---------|-----------|-------------|
| **Ahultasuna** | Vulnerabilidad | Defecto o debilidad en un sistema o aplicación que un atacante puede explotar. |
| **Aktiboa** | Activo | Cualquier elemento físico o digital que tiene valor para el centro: servidores, datos, software, personas… |
| **Arriskua** | Riesgo | Probabilidad de que una amenaza explote una vulnerabilidad causando un daño, y el impacto de dicho daño. |
| **Autentikazio-faktore anitz (MFA)** | Autenticación multifactor | Mecanismo de seguridad que exige dos o más factores para verificar la identidad de un usuario. |
| **Babeskopia** | Copia de seguridad | Copia de los datos para recuperarlos en caso de pérdida del original. |
| **Ciberseguridad** | Zibersegurtasuna | Ciberseguridad (término en castellano como referencia). |
| **DPD (Datuen Babeserako Ordezkaria)** | DPD (Delegado de Protección de Datos) | Figura exigida por el RGPD: persona que supervisa la protección de datos en una organización. |
| **EIPD (Datuen Babesari buruzko Eraginaren Ebaluazioa)** | EIPD / DPIA | Proceso para evaluar los riesgos que un tratamiento de datos puede suponer para los derechos y libertades de las personas (RGPD). |
| **ENS (Esquema Nacional de Seguridad)** | ENS | Normativa española que regula la seguridad de los sistemas de información del sector público (RD 311/2022). |
| **Equipo Eraldi** | Eraldi taldea | Equipo de transformación digital de los centros educativos, impulsado por Tknika. |
| **Erresilientzia** | Resiliencia | Capacidad de un sistema para volver a la normalidad y seguir funcionando tras un incidente. |
| **Firewall (suebakia)** | Cortafuegos | Hardware o software que filtra el tráfico de red, aceptando solo las conexiones permitidas. |
| **Gobernantza** | Gobernanza | Conjunto de procesos y estructuras que definen las decisiones, roles y políticas para la gestión de la ciberseguridad. |
| **Intzidentzia** | Incidente | Evento que compromete la seguridad: filtración de datos, ataque de ransomware, acceso no autorizado… |
| **ISO/IEC 27001** | ISO/IEC 27001 | Estándar internacional para la gestión de la seguridad de la información (SGSI). |
| **LOPDGDD** | LOPDGDD | Ley Orgánica de Protección de Datos Personales y Garantía de Derechos Digitales (LO 3/2018). |
| **Magerit** | Magerit | Metodología de análisis de riesgos desarrollada por el CCN-CERT, utilizada para el cumplimiento del ENS. |
| **Mehatxua** | Amenaza | Evento que puede causar un daño potencial a sistemas o datos, de forma intencionada o accidental. |
| **NIS2** | NIS2 | Directiva de Ciberseguridad de la Unión Europea (2022/2555), que regula las medidas de seguridad en sectores críticos. |
| **NIST CSF 2.0** | NIST CSF 2.0 | Marco de Ciberseguridad publicado por el NIST (EE.UU.), útil para organizaciones de todos los sectores. |
| **Pasahitzen kudeatzailea** | Gestor de contraseñas | Aplicación que crea, almacena y autocompleta contraseñas de forma segura (Bitwarden, KeePass…). |
| **Phishing** | Phishing | Técnica para robar datos personales (contraseñas, datos bancarios) mediante correos, SMS o webs falsas. |
| **Matriz RACI** | RACI matrizea | Matriz para definir roles y responsabilidades: Responsible, Accountable, Consulted, Informed. |
| **Ransomware** | Ransomware | Software malicioso que cifra los archivos y exige un rescate a cambio de su recuperación. |
| **RGPD (DBEO)** | RGPD | Reglamento General de Protección de Datos de la Unión Europea (2016/679). |
| **RTO** | RTO (Recovery Time Objective) | Tiempo máximo aceptable para recuperar un servicio tras un incidente. |
| **Segurtasun-politika** | Política de seguridad | Documento que define el enfoque de seguridad, las normas y las responsabilidades de una organización (ENS Art. 12). |
| **SIEM** | SIEM | Sistema de gestión centralizada de eventos de seguridad (recopila registros, los analiza y genera alertas). |
| **VLAN** | VLAN | Tecnología para segmentar la red local, separando el tráfico y los accesos en zonas lógicas diferentes. |
| **WiFi** | WiFi | Red de área local inalámbrica (WLAN). |
| **Zifraketa** | Cifrado | Técnica para convertir los datos en ilegibles; solo quien tiene la clave puede descifrarlos. |
| **3-2-1 araua** | Regla 3-2-1 | Principio básico de copias de seguridad: 3 copias, 2 soportes diferentes, 1 ubicación física distinta. |
