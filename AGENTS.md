# AGENTS.md — Zik project instructions for AI coding agents

## Development environment (Docker)

This project runs entirely inside Docker containers. **Do not run `npm install`, `uv sync`, `uv run`, or any build commands directly on the host.** Always use `docker compose`.

### Services

| Service | Command | Purpose |
|---------|---------|---------|
| **dev** | `docker compose up dev` | Astro dev server with HMR on `http://localhost:4321/zik/` |
| **templates** | `docker compose run --rm templates` | Generate DOCX/PDF templates from Markdown sources (one-shot) |

### Why Docker for everything

- The Astro dev server runs inside a `node:22-slim` container with a Docker-managed `node_modules` volume — avoids root-owned files and host permission issues.
- Template generation runs inside an `uv` container that installs `pandoc` and Python dependencies (weasyprint, markdown-it-py, python-docx) at runtime. Running `uv` on the host would create root-owned `.venv` directories and output files that break host access.
- `public/templates/` files generated by the templates container will be root-owned — this is expected and handled by Docker volume mounts.

### Template generation

Templates are authored as Markdown in `scripts/templates-md/`. Each uses `{{placeholder}}` syntax for fields that get populated from user data (localStorage on the client side, or a JSON file at build time).

```bash
# Regenerate all templates (Basque + Spanish)
docker compose run --rm templates

# After running, root-owned files in public/templates/ are normal.
# To clean them if needed: sudo rm -f public/templates/*
```

At build time (no `--data` flag), all `{{placeholders}}` are replaced with visual gaps (underline lines in PDF, underscores in DOCX). When a `--data file.json` is provided, filled values appear and empty values remain as gaps.

### Node dependencies

```bash
# Install or update npm packages
docker compose run --rm dev npm install
docker compose run --rm dev npm install <package>
```

The `node_modules` volume is Docker-managed — never run `npm` on the host.

```
zik/
├── compose.yaml              # Docker services
├── astro.config.mjs          # Astro + Starlight config

│   ├── content/docs/es/      # Spanish mirror
│   ├── components/           # Astro components
│   ├── styles/               # CSS
│   └── content.config.ts     # Starlight content config
├── scripts/
│   ├── templates-md/         # Markdown template sources
│   │   └── es/               # Spanish templates
│   ├── generate_templates.py # Template generation (DOCX + PDF)
│   └── pyproject.toml        # Python dependencies (uv)
├── public/
│   └── templates/            # Generated DOCX/PDF files (gitignored? check .gitignore)
└── sources/                  # Reference documents (NIST, ISO, RGPD, etc.)
```

## Key decisions

- **Basque-first**: root locale (`/`) is Basque, Spanish at `/es/`
- **Client-side only**: user data stored in `localStorage`, no backend
- **No host-level package managers**: always use `docker compose` for `npm` and `uv` commands
- **Template placeholders**: `{{fieldName}}` in Markdown sources → visual gaps at build time, filled values with `--data`
